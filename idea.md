# [기획 및 설계 문서] BidVett (가칭)

## 1. 프로젝트 개요 및 주 타겟층
* **프로젝트 명:** BidVett (Upwork 프리랜서용 매칭 및 리스크 스크리닝 서비스)
* **프로젝트 목적:** 2026년 현재 급등한 Upwork 지원 비용(Connects) 부담을 최소화하고, AI 스팸 지원서 범람 속에서 프리랜서가 실제 '면접(Interview)' 및 '고용(Hire)'으로 이어질 확률이 가장 높은 핵심 공고만 타겟팅할 수 있도록 돕는 '방어형(비용 절감) SaaS' 개발.
* **주 타겟층:**
    * **핵심 타겟:** Upwork에서 활동하는 글로벌 1~5년 차 주니어/미드 레벨 프리랜서 (특히 지원 비용 대비 수주율이 낮아 자본 압박을 크게 느끼는 계층).
    * **확장 타겟:** 대량의 공고 분석 시간이 부족한 탑레이티드(Top Rated) 고단가 프리랜서.
* **서비스 언어:** 본 프로젝트의 타겟층은 외국 영어권 사용자이므로, 모든 UI/UX 텍스트, 랜딩 페이지 카피, 마케팅 문구, AI 출력 메시지 등 사용자에게 노출되는 모든 산출물은 **영문(English)으로 작성**되어야 한다.

---

## 2. 개발 및 배포 기술 스택
초기 개발 속도와 1인 창업(바이브 코딩) 생산성을 극대화하기 위해 다음과 같은 글로벌 검증 스택을 채택합니다.

* **Frontend:** Next.js (App Router), Tailwind CSS, shadcn/ui (UI 컴포넌트 생산성 극대화)
* **Backend / Database:** Supabase (PostgreSQL, Auth, 및 Realtime DB 인프라를 일원화하여 개발 속도 최적화)
* **AI Engine:** OpenAI API (`gpt-4o-mini`를 메인 분석기로 사용해 비용 절감, 정밀 최종 판독 시 필요에 따라 `gpt-4o` 스위칭 설계)
* **Payment:** Stripe (글로벌 소액 결제 및 정기 구독 대응)
* **Deployment:** Vercel (Frontend & Serverless Functions), Supabase Cloud

## 3. 알고리즘 및 이중 위험도(Risk) 판독 구조 세부 설계
본 서비스의 핵심 가치인 '리스크 판독'은 시스템(백엔드)의 정량 평가와 AI의 정성 평가가 결합된 '이중 스크리닝' 체계로 작동합니다.

### 3.1 이중 리스크 산출 방식 및 의미 정의

#### ① 백엔드 시스템 위험도 (정량적 하드웨어 필터링)
* **의미:** 클라이언트가 업워크 플랫폼 상에서 남긴 '과거 객관적 수치 데이터'만을 기준으로 기계적으로 판독하는 위험도입니다. 사기꾼들이 수치를 조작하기 어렵기 때문에, 가장 확실한 '1차 컷탈락' 기준이 됩니다.
* **산출 방식 (Rule-based):** 다음 조건 중 하나라도 해당하면 시스템이 즉시 `CRITICAL_RISK` 플래그를 발생시킵니다.
    * 클라이언트의 과거 고용률(Hire Rate)이 20% 미만인 경우 (공고만 올리고 먹튀하는 유령 공고).
    * 결제 수단 미인증 (Payment Unverified) 상태이면서 과거 지출 총액이 $0인 경우.
    * 과거 고용했던 프리랜서들이 남긴 평점 평균이 3.5 이하인 경우.

#### ② AI 맥락 위험도 (`risk_level` - 정성적 소프트웨어 필터링)
* **의미:** 수치상으로는 정상(예: 고용률 90%, 결제 인증 완료)처럼 보이지만, 구인 공고 본문(텍스트)의 뉘앙스, 독소 조항, 사기 유도 패턴을 AI가 채용 전문가의 시선으로 추론한 위험도입니다.
* **산출 방식 (LLM Contextual Reasoning):** OpenAI API가 본문을 분석하여 3단계 등급으로 추론합니다.
    * `SAFE`: 요구사항이 명확하고 플랫폼 가이드를 준수하는 정상 공고.
    * `WARNING`: 사기는 아니지만 무리한 마감일 설정, 과도한 무료 샘플 요구, 단가 후려치기 뉘앙스가 감지됨.
    * `DANGER`: "텔레그램으로 연락 달라", "보증금을 먼저 입금해라", "리뷰 조작 알바다" 등 문맥상 100% 사기 및 플랫폼 위반으로 추론됨.

### 3.2 매칭 스코어링 알고리즘 (Match Score)
위험 요소를 통과한 공고에 한해 유저의 프로필(이력서)과 구인 공고 본문을 비교 분석합니다.
* **분석 매트릭스 및 가중치:**
    1. **기술 일치도 (Technical Fit - 40%):** 요구 스킬셋 일치 여부 및 숙련도 매칭.
    2. **예산/단가 일치도 (Budget Fit - 30%):** 클라이언트가 제시한 금액(또는 과거 평균 지출 단가)과 프리랜서의 희망 단가 갭 분석.
    3. **커뮤니티 성향 매칭 (Context Fit - 30%):** 클라이언트가 선호하는 국가/시간대 조율 가능 여부.
* **최종 점수 표기:** 1~100점의 `Connect Value Score` 산출.

### 3.3 최종 대시보드 화면 표기 로직 (UX)
* 백엔드 위험도가 `CRITICAL_RISK`이거나 AI의 `risk_level`이 `DANGER`인 경우, 서비스는 유저에게 **[지원 절대 금지]** 경고를 크게 띄우며 소중한 Connects 소모를 막아줍니다. 두 위험도가 모두 안전할 때만 최종 점수와 지원 전략을 화면에 노출합니다.

## 4. AI의 역할 정의 및 프롬프트 최적화

### 4.1 AI의 역할 특정 (System Persona)
* AI는 단순히 글을 요약하는 비서가 아니라, **"5년 경력의 글로벌 업워크(Upwork) 전문 채용 헤드헌터이자 프리랜서 리스크 관리 컨설턴트"**로 역할을 한정합니다. 
* AI의 출력은 철저히 정량적 점수(Score)와 냉정한 판단(Reason), 그리고 실행 가능한 액션 플랜(Action Plan)의 구조화된 JSON 데이터로만 반환되도록 강제합니다.

### 4.2 시스템 프롬프트(System Prompt) 설계 가이드라인
```text
너는 업워크(Upwork) 생태계를 완벽하게 이해하고 있는 5년 경력의 전문 프리랜서 매칭 컨설턴트야. 너의 목표는 프리랜서의 프로필과 구인 공고를 대조하여, 프리랜서가 소중한 'Connects(지원 토큰)'를 낭비하지 않도록 냉정하게 판독하는 것이다.

[출력 포맷]
반드시 다음 JSON 형식을 엄격히 준수하여 응답해라:
{
  "risk_level": "SAFE" | "WARNING" | "DANGER",
  "contextual_red_flags": ["위험 요인 문장이나 이유를 배열로 작성, 없을 시 빈 배열"],
  "match_score": 1-100 정수,
  "score_reason": "점수 산정 이유 요약",
  "action_tip": "지원서에 쓸 핵심 무기 가이드라인"
}
```

## 5. 유저 플로우 (사용성) 및 사용자 로그인
1. **회원가입 및 인증:** Supabase Auth를 통한 '구글 소셜 로그인(Google OAuth)' 단일창 구현.
2. **프로필 저장:** 최초 로그인 후 유저는 자신의 이력서 및 포트폴리오 텍스트 요약본을 서비스에 등록/저장.
3. **공고 분석 수행:** Upwork에서 마음에 드는 공고를 발견하면 공고 전체 페이지를 복사하여 BidVett 분석창에 붙여넣고 [분석하기] 버튼 클릭.
4. **리포트 발행:** 3초 이내에 이중 위험도 스크리닝 및 매칭 점수 리포트 팝업.

## 6. 테스트 시나리오 및 소액 즉시 수익화 로드맵
인맥이나 별도의 단톡방이 없는 환경을 극복하기 위해, 철저히 글로벌 타겟 커뮤니티 마케팅과 MVP 단계부터 즉시 소액 현금을 확보하는 린(Lean) 성장 모델을 따릅니다.

### 1단계: 랜딩 페이지 + 맛보기 기능 오픈 (1~2주 차)
* 웹사이트에 Stripe 결제 모듈과 Supabase를 첫날부터 연동하여 배포.
* 로그인한 유저에게 '평생 딱 3회 무료 분석권'만 제공. 소진 즉시 "$0.99 단건 결제" 또는 "일주일 무제한 패스 = $4.99" 결제창 유도.

### 2단계: 레딧(r/upwork) 기반 무자본 트래픽 유입 (3~4주 차)
* 레딧에서 조언을 구하는 유저들의 글을 모니터링하여 공고 본문을 BidVett 백엔드 엔진에 돌린 리포트를 댓글로 정성스럽게 제공.
* 댓글 하단에 3회 무료 체험 링크를 남겨 타겟 트래픽 획득.

### 3단계: 소액 결제를 통한 초기 생존 및 유료 구독(SaaS) 확장 (5주 차 이후)
* 단건 결제 매출로 AI API 비용 상쇄 및 지속 유입되는 찐 팬 유저들을 식별하여 "월 $19 정기 구독 플랜" 확장.

## 7. 초기 유저 테스트 후 확장성 고려 사항 (Architecture Scalability)
* **수동 입력에서 자동화로 전환:** 초기에는 크롤링 제재 회피를 위해 '복사·붙여넣기' 고수.
* **크롬 익스텐션 이식 고려:** 추후 웹사이트 스크롤 시 오버레이 창으로 리포트를 띄워주는 '크롬 익스텐션'으로 쉽게 이식될 수 있도록 입력 핸들러 모듈을 프론트엔드단과 분리 설계.
