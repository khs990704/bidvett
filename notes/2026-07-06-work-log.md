## 2. 기존 계정 삭제 후 신규 가입 테스트 절차 정리

### 권장 절차
테스트 목적으로 같은 Google 계정을 새 가입처럼 만들려면 Supabase Dashboard에서 Auth user를 삭제.

1. Supabase Dashboard → Authentication → Users
2. 테스트 이메일 검색
3. 해당 user 삭제
4. 브라우저 로그아웃 또는 쿠키 삭제
5. Google 로그인 재시도
6. `credit_ledger`에서 `free_grant +5` 확인

### 근거
`users_profile`, `analyses`, `credit_ledger`, `subscriptions`는 `auth.users(id) ON DELETE CASCADE`로 연결되어 있어 Auth user 삭제 시 앱 데이터도 같이 정리된다.

## 3. 주간·월간 구독 취소 기능 점검 및 보강

### 기존 상태
월간 구독 취소 API와 UI는 이미 존재.

- `src/app/api/subscription/cancel/route.ts`
- `src/app/account/AccountBilling.tsx`

취소 시 Dodo에 다음 결제 취소 요청:

```ts
dodoClient().subscriptions.update(subId, {
  cancel_at_next_billing_date: true,
});
```

그리고 우리 DB에는 `cancelled_at`만 기록한다. `status`는 즉시 `canceled`로 바꾸지 않으므로 현재 paid period는 유지된다.

### 문제
주간 플랜이 Dodo에서 recurring weekly 상품이면 자동 갱신이 발생하는데, 기존 UI/API 응답은 월간만 `cancel_at_period_end`를 노출했다.

### 조치
주간 recurring 구독도 취소 가능하도록 보강.

변경 파일:
- `src/lib/types/api.ts`
- `src/app/api/credits/route.ts`
- `src/app/account/AccountBilling.tsx`
- `src/components/analyze/CreditBadge.tsx`
- `src/app/pricing/page.tsx`

핵심 변경:
- `active_pass`에 `is_recurring`, `cancel_at_period_end` 추가
- 주간 recurring 플랜도 Account → Billing에서 `Cancel subscription` 버튼 노출
- 취소 예정 주간 플랜에 `cancels at period end` 표시
- Pricing 주간 문구를 `Renews weekly, cancel anytime`으로 변경

### 정책

```text
주간 recurring 구독: 취소 가능, 현재 7일 기간은 유지, 다음 주 결제 차단
월간 recurring 구독: 취소 가능, 현재 30일 기간은 유지, 다음 달 결제 차단
주간 one-time 패스: 자동 갱신이 없으므로 취소 대상 아님
```

### 운영 주의
Dodo Dashboard에서 `weekly_pass_499`가 반드시 **recurring weekly** 상품이어야 주간 구독 취소 흐름이 의미가 있다. one-time product이면 자동 갱신 자체가 없다.

## 4. 중복 recurring 구독 결제 차단

### 문제
기존 `subscription.active` webhook은 새 구독 row를 insert하기 전에 DB의 다른 active row를 `status='canceled'`로 정리했다. 하지만 Dodo 측 기존 구독을 실제로 취소하지는 않으므로, 주간→월간 또는 월간→주간 중간 변경 시 Dodo에서 두 구독이 모두 살아 이중 청구될 위험이 있었다.

### 정책 결정
MVP에서는 proration, 즉시 업그레이드/다운그레이드, 남은 기간 정산을 지원하지 않는다.

따라서 active 주간/월간 구독이 있으면 새 recurring checkout을 차단한다.

```text
기존 주간 구독 중 → 월간 구독 결제 시도: 409 차단
기존 월간 구독 중 → 주간 구독 결제 시도: 409 차단
단건 크레딧 구매: 기존 구독과 무관하게 허용
```

### 조치
`src/app/api/checkout/route.ts`에 active subscription guard 추가.

조건:
- `plan !== 'credit_single'`
- `subscriptions.status = 'active'`
- `period_end > now()`
- `plan in ('weekly_pass', 'monthly_sub')`

차단 응답:

```text
409 ERR_BAD_REQUEST
reason = active_subscription_exists
```

## 6. 환경 변수 이름 주의

### 발견
문서 일부는 `NEXT_PUBLIC_DODO_PRODUCT_*`를 언급하지만, 실제 코드가 읽는 값은 다음이다.

```text
NEXT_PUBLIC_DODO_PRICE_SINGLE
NEXT_PUBLIC_DODO_PRICE_WEEKLY
NEXT_PUBLIC_DODO_PRICE_MONTHLY
```

### 운영 주의
Vercel에는 코드 기준 env 이름인 `NEXT_PUBLIC_DODO_PRICE_*`를 넣어야 한다. 특히 `NEXT_PUBLIC_DODO_PRICE_WEEKLY`는 Dodo의 recurring weekly product id를 가리켜야 한다.
